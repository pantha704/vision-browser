# Phase 11, Plan 01: Model JSON Validation Pipeline

## Goal
Add `ModelResponseError` exception class and JSON validation pipeline that validates every vision model response before use.

## Requirements
- MODEL-01: Every vision model response validated as JSON before use
- MODEL-04: Structured error reporting includes expected schema, raw response, debugging context

## Implementation

### 1. Add `ModelResponseError` to exceptions.py
```python
class ModelResponseError(VisionAPIError):
    """Model response could not be parsed as valid JSON."""
    def __init__(self, message: str, raw_response: str, expected_schema: dict | None = None):
        super().__init__(message)
        self.raw_response = raw_response
        self.expected_schema = expected_schema
        self.context: dict[str, str] = {}

    def with_context(self, **kwargs) -> "ModelResponseError":
        self.context.update(kwargs)
        return self
```

### 2. Add `_validate_json_response` method to VisionClient
- Takes raw text and optional schema
- Attempts JSON extraction via existing `_extract_json`
- On failure, raises `ModelResponseError` with raw response, schema, context
- Returns extracted JSON dict on success

### 3. Integrate into `_nim_analyze` and `_groq_analyze`
- After getting text content, call `_validate_json_response`
- Pass schema for context in error reporting
