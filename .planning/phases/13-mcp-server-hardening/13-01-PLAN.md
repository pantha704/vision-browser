# Phase 13, Plan 01: MCP Server Hardening

## Goal
Add health check, error recovery, connection state tracking, and structured error responses to MCPServer.

## Requirements
- MCP-01: Health check tool (`ping`/`health`) for client verification of server status
- MCP-02: All MCP tools wrapped in error recovery — never raise unhandled exceptions
- MCP-03: Connection state tracking (connected, disconnected, recovering) with state queries
- MCP-04: Structured error responses with retry-after hints on model API failures

## Implementation

### 1. Add ConnectionState enum and health tool
```python
class ConnectionState(Enum):
    CONNECTED = "connected"
    DISCONNECTED = "disconnected"
    RECOVERING = "recovering"
    DEGRADED = "degraded"
```

### 2. Health tool
- Add `HEALTH_TOOL` to ALL_TOOLS
- Returns: `{status: "ok"|"degraded"|"error", state: str, tools_available: [...], uptime: float}`

### 3. Connection state tracking
- Track state on orchestrator attach/detach
- Update state on tool failures (transition to RECOVERING, then DEGRADED if persistent)
- `_state` attribute queryable via health tool

### 4. Error recovery wrapper
- Wrap all tool handlers in try/except
- Catch exceptions and return structured error: `{error: str, type: str, retry_after: int|None}`
- For VisionAPIError (model failures): include retry_after hint
- For BrowserError: suggest reconnecting browser

### 5. Structured error responses
- Consistent format: `{"success": false, "error": "...", "error_type": "...", "retry_after": N|null, "suggestion": "..."}`

## Verification
- Health tool returns correct status
- All tools return structured errors on failure
- Connection state transitions correctly
- No unhandled exceptions propagate to clients
- Existing tool response shapes unchanged for success cases
