# Phase 12, Plan 01: Differential Screenshot Integration into FastOrchestrator

## Goal
Wire `DifferentialScreenshot` into `FastOrchestrator._run_loop()` with config toggle, storage, and cleanup.

## Requirements
- DIFF-01: Differential screenshots automatically captured before and after each action (opt-in via config)
- DIFF-02: Configurable diff threshold to skip identical or near-identical frames
- DIFF-03: Diff results stored with task context and retrievable for debugging
- DIFF-04: Automatic cleanup of old diffs (keep last N per session, configurable)

## Implementation

### 1. Config: Add `auto_diff_screenshots` and `diff_max_retain` to config.py
```python
# In OrchestratorConfig:
auto_diff_screenshots: bool = False  # Default off
diff_threshold: float = 0.01  # Pixel difference ratio
diff_max_retain: int = 10  # Keep last N diffs per session
```

### 2. FastOrchestrator: Integrate DifferentialScreenshot
- Initialize `DifferentialScreenshot` in `__init__` when enabled
- In `_run_loop()`:
  - Before vision analysis: check if screenshot changed from previous
  - After action execution: capture another diff
  - Store diff metadata in session context (timestamp, action, path)
- On `close()`: cleanup old diffs

### 3. Diff storage: Simple JSON manifest in session directory
- `.vision-browser/sessions/{session}/diffs/manifest.json`
- Each entry: `{timestamp, action, before_path, after_path, changed: bool, change_ratio: float}`

### 4. Cleanup: Remove oldest diffs beyond `diff_max_retain`

## Verification
- Tests for config defaults and overrides
- Tests for diff capture in orchestrator loop
- Tests for cleanup behavior
- Benchmark: auto-capture adds <200ms per action
